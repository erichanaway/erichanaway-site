Data Quality Manager

A browser-based application that analyzes HMIS data exports, identifies data quality issues, and helps agencies improve reporting through an expanding library of automated validation rules.

View Live Manager View GitHub

Project Overview

The Data Quality Manager is a browser-based tool designed to help HMIS administrators and agencies identify data quality issues before submitting reports. Users upload an Excel workbook exported from HMIS, and the manager evaluates each record against a growing library of validation rules.

Instead of manually reviewing rows one by one, the manager quickly identifies missing information, invalid project data, incomplete client records, and other common issues. Its goal is to reduce reporting errors, improve data quality, and save staff review time.

Key Features

Upload and analyze HMIS Excel workbooks
Automated rule-based validation engine
Data quality dashboard with KPI summary cards
Detailed flag report with severity, description, and recommended resolution

Technologies Used

HTML, CSS, JavaScript (ES6), Vite, SheetJS (XLSX), Git, GitHub, and Vercel.

Challenges and Solutions

One of the main challenges was designing a validation system that could grow over time. Instead of writing all data quality checks directly inside the dashboard code, I created a separate rule library where each rule contains its own ID, category, severity, message, resolution, and validation logic.

This structure allows the Data Quality Manager to expand from a few basic checks to hundreds of validation rules without rewriting the main application. The result is a more scalable and maintainable system for reviewing HMIS data.
